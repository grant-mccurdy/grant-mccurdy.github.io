Project Brief

Assessment Intelligence

Public-safe assessment system work: SQL-backed extracts, analytics, reproducible R processing, dashboarding, diagnostics, and decision-support reporting for mathematics programs.

Open Dashboard GitHub Repo

Overview

From assessment design to decision support

This project demonstrates how a department or school can move from assessment design to analysis and reporting without exposing real student records. The public analytics layer uses five SQL-backed synthetic extracts with 442 aggregate rows, and keeps the core data-generation and warehouse foundation in synthetic-education-data.

Portal Notes

What this project proves

Static dashboard deployment

The dashboard runs on GitHub Pages from the same assessment-intelligence SQL extract data used by the public report artifacts, with browser-side aggregation, SVG charts, and no external JavaScript dependencies.

Public-safe synthesis

The assessment repo stages analytics and reporting workflows on public-safe synthetic outputs, while the separate synthetic math department repo owns the reusable generation and validation story.

SQL-backed analysis

The repo now extracts course-section performance, assignment growth, non-participation patterns, LMS reconciliation, and 287 student readiness records from the synthetic DuckDB warehouse.

Hosted serving path

The optional Supabase extract path demonstrates a public-safe serving layer while keeping DuckDB as the reproducible local baseline.

AI-assisted review

Dry-run OpenAI review artifacts summarize the five deterministic SQL extracts without querying Supabase directly or transforming source CSVs.

Visualization catalog

The source repo now includes raincloud, hexbin, and calibration plot patterns for assessment decision questions.

Inspect Next

SQL, hosted extracts, and review artifacts

The assessment layer now has a stronger analytics contract: deterministic DuckDB extracts first, optional Supabase serving views second, and AI-assisted narrative review last. This keeps extraction, serving, and interpretation separate.

Safety Boundary

Synthetic by design

Public artifacts must not include real student names, emails, IDs, grades, rosters, LMS exports, submissions, or private school reporting artifacts. Canvas is treated as one possible adapter, not the project identity.

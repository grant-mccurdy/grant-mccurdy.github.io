Project Brief

Statistical Risk Modeling in R

Public-safe business analytics project that turns synthetic account-risk records into an interpretable probability model, validation diagnostics, calibration checks, lift analysis, threshold economics, and executive-ready decision notes.

GitHub Repo Executive Brief Model Card

Risk Modeling Surface

Interpretable model, operational decision support

The project uses base R to generate synthetic B2B account-risk data, compare candidate models, validate holdout performance, quantify uncertainty, and translate model output into human-review prioritization guidance.

Evidence Packet

  • 1,400-row synthetic account-risk dataset
  • Logistic regression / GLM probability modeling
  • Repeated stratified cross-validation
  • Flexible spline benchmark comparison
  • Holdout AUC, log loss, and Brier score
  • Bootstrap metric uncertainty intervals
  • ROC, lift, calibration, and subgroup diagnostics
  • Threshold economics and risk-category interpretation
  • Executive brief, model card, methodology, and public-safety notes

Validation

What reviewers can inspect

Model Selection

Candidate models are compared against validation metrics, with a spline benchmark used as a stress test before choosing the interpretable operating model.

Decision Metrics

The executive brief reports holdout AUC 0.719, log loss 0.422, Brier score 0.129, top-decile lift 2.71x, and threshold-level operating tradeoffs.

Calibration And Lift

The evidence packet includes calibration diagnostics, decile lift, subgroup calibration, sensitivity checks, and scenario profiles for communication.

Public-Safe R Pipeline

The repo uses synthetic records only, requires no packages or network access, and includes validation scripts that check for private-source leakage.

Inspect In Source Repo

Reproducible R evidence path

The project is intentionally lightweight: R scripts generate data, fit models, render reports, build figures, and validate the public boundary from a `make all` pipeline.

Safety Boundary

Synthetic records, human review

The project is not a real account-scoring system and is not for automated customer, credit, employment, clinical, or private-data decisions. It is a public-safe demonstration of statistical modeling judgment, validation, and decision-support communication.
